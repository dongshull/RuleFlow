# USG-LEGO 后端接口文档

本文档详细描述了 USG-LEGO 项目的所有后端接口，包括请求方式、URL、参数、响应格式等信息，供前端开发人员对接使用。

## 基础信息

- **基础URL**: `http://localhost:8080` (默认端口，可通过 LISTEN 环境变量配置)
- **请求格式**: JSON
- **响应格式**: JSON
- **认证方式**: API Key 认证

## 全局响应状态码

| 状态码 | 说明             |
|--------|------------------|
| 200    | 请求成功         |
| 400    | 请求参数错误     |
| 401    | 未授权/认证失败  |
| 403    | 禁止访问         |
| 404    | 资源不存在       |
| 500    | 服务器内部错误   |

## 接口列表

### 1. 健康检查接口

#### 接口说明
检查服务运行状态

#### 请求URL
```
GET /health
```

#### 请求参数
无

#### 响应示例
```json
ok
```

#### 响应说明
- 响应内容固定为 "ok"
- HTTP状态码: 200

### 2. 获取文件内容

#### 接口说明
获取指定路径的文件内容，支持文件流和JSON两种响应格式

#### 请求URL
```
GET /api/files
```

#### 请求参数
| 参数名 | 类型   | 必填 | 说明                                  |
|--------|--------|------|---------------------------------------|
| path   | string | 是   | 文件路径                              |
| api    | string | 否   | API密钥，访问私有文件时必需           |

#### 请求头
| 参数名          | 值                    | 说明                                       |
|-----------------|------------------------|--------------------------------------------|
| Accept          | application/json       | 指定返回JSON格式的文件信息                 |
| Accept          | application/octet-stream | 指定返回文件流（默认）                    |

#### 响应格式（文件流）
当请求头 Accept 不为 application/json 时返回文件流：

```
Content-Type: application/octet-stream
Content-Disposition: attachment; filename="filename.ext"

[文件内容]
```

#### 响应格式（JSON）
当请求头 Accept 为 application/json 时返回文件元数据：

```json
{
  "path": "/rules/default.list",
  "isDir": false,
  "fileInfo": {
    "name": "default.list",
    "isDir": false,
    "size": 85,
    "modTime": "2025-08-30T19:38:39.93346734+08:00"
  }
}
```

#### 响应格式（目录列表）
当请求路径为目录时返回目录内容列表：

```json
{
  "path": "/rules",
  "isDir": true,
  "files": [
    {
      "name": "default.list",
      "isDir": false,
      "size": 85,
      "modTime": "2025-08-30T19:38:39.93346734+08:00"
    }
  ]
}
```

#### 错误响应
```json
{
  "error": "错误信息"
}
```

#### 错误信息说明
| 错误信息           | HTTP状态码 | 说明                           |
|--------------------|------------|--------------------------------|
| Path parameter is required | 400        | 未提供path参数                |
| Invalid path       | 403        | 路径包含非法字符或路径遍历攻击 |
| File not found     | 404        | 文件或目录不存在               |
| API key required   | 401        | 访问私有文件未提供API密钥      |
| Invalid API key    | 401        | 提供的API密钥无效              |
| Server configuration error | 500 | 服务器配置错误                |
| Failed to access file | 500     | 服务器无法访问文件            |
| Failed to read directory | 500  | 服务器无法读取目录            |

#### 接口说明
直接访问原始文件内容

#### 请求URL
```

```

#### 请求参数
| 参数名 | 类型   | 必填 | 说明                        |
|--------|--------|------|-----------------------------|
| api    | string | 否   | API密钥，访问私有文件时必需 |

#### 响应示例
```
Content-Type: text/plain; charset=utf-8

Raw file access placeholder
```

#### 响应说明
- 返回文件原始内容

### 4. 获取API密钥列表

#### 接口说明
获取所有API密钥列表（需要管理员权限）

#### 请求URL

#### 请求参数
| 参数名 | 类型   | 必填 | 说明       |
|--------|--------|------|------------|

#### 响应示例
```json

    "id": 1,
    "token": "32字节随机字符串",
```

### 5. 创建API密钥

#### 接口说明
生成新的API密钥（需要管理员权限）

#### 请求URL
```
POST /api/keys
```

#### 请求参数
| 参数名 | 类型   | 必填 | 说明       |
|--------|--------|------|------------|
| api    | string | 是   | 管理员密钥 |

#### 请求体示例
```json

  "token": "可选，自定义密钥，不提供则自动生成",

}
```

#### 响应示例
```json
{
  "id": 2,
  "token": "新生成的32字节随机字符串",
  "note": "用于iOS设备的密钥",
  "created_at": "2025-08-30T19:38:39Z"
}
```

#### 响应说明
- HTTP状态码: 200

#### 响应说明

## 权限控制说明

### 公共路径
- 默认以 `/public` 开头的路径为公共路径
- 可通过配置文件 `.usg-lego.yml` 自定义公共路径规则
- 公共路径下的文件可无需认证直接访问

### 私有路径
- 除公共路径外的所有路径均为私有路径
- 访问私有路径需要提供有效的API密钥

### API密钥配置
- 生产环境建议通过环境变量 `API_KEY` 设置初始API密钥
- 系统使用SQLite数据库存储API密钥，支持动态管理
- API密钥验证使用数据库查询验证
- 支持多个有效API密钥同时存在

## 安全机制

1. **路径遍历防护**: 所有路径参数都经过 `filepath.Clean` 处理，并检查不能跳出 ROOT_DIR
2. **API密钥验证**: 私有文件访问需要提供有效的API密钥，使用数据库查询验证
3. **访问控制**: 写操作和私有路径读操作都必须携带有效的API密钥
4. **密钥生成**: 新创建的API密钥使用加密安全的随机数生成器创建

## 环境变量配置

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| ROOT_DIR | ./configs | 配置文件根目录 |
| LISTEN | :8080 | 监听地址 |
| LOG_LEVEL | info | 日志级别 |
| API_KEY | 无 | 初始API密钥（仅在数据库为空时使用） |

## 路径规则配置

可通过在 `ROOT_DIR` 目录下创建 `.usg-lego.yml` 文件来配置公共和私有路径规则：

```yaml
public:
  - "/public/**"
  - "/assets/**"

private:
  - "/private/**"
  - "/conf/**"
```

## 数据库存储

API密钥现在存储在SQLite数据库中，确保数据持久化和安全性：
- 数据库文件位置：`ROOT_DIR/usg-lego.db`
- 表结构：包含id、token、note和created_at字段
- 初始化：系统启动时自动创建数据库和表结构
- 默认密钥：如果数据库为空且设置了API_KEY环境变量，则会创建初始密钥

## 错误处理

所有错误响应都遵循以下格式：
```json
{
  "error": "错误信息"
}
```

## 示例请求

### 获取公共文件（无需API密钥）
```bash
curl "http://localhost:8080/api/files?path=/public/public.conf"
```

### 获取私有文件（需要API密钥）
```bash
curl "http://localhost:8080/api/files?path=/private.rules&api=您的API密钥"
```

### 获取文件元数据（JSON格式）
```bash
curl -H "Accept: application/json" "http://localhost:8080/api/files?path=/modules/module1.conf&api=您的API密钥"
```

### 获取目录列表
```bash
curl -H "Accept: application/json" "http://localhost:8080/api/files?path=/rules&api=您的API密钥"
```

### 获取API密钥列表（需要管理员权限）
```bash
curl "http://localhost:8080/api/keys?api=您的管理员API密钥"
```

### 创建新的API密钥（需要管理员权限）
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"note":"新密钥备注"}' \
  "http://localhost:8080/api/keys?api=您的管理员API密钥"
```

### 删除API密钥（需要管理员权限）
```bash
curl -X DELETE "http://localhost:8080/api/keys/1?api=您的管理员API密钥"
```

### 生产环境启动示例
```bash
ROOT_DIR=./configs LISTEN=:8080 API_KEY=your-initial-api-key go run cmd/server/main.go
```
